## Invention Lab 2026

Invention Lab (ER-IN-2601) is a 5-day immersive STEM training program for Sea Cadets that introduces the fundamentals of embedded systems, sensor integration, 3D design, IoT communication, and project-based engineering.

The program is designed for students with **no prior experience** in coding or electronics. By the end of the week, each team will have built a working capstone project that combines firmware, hardware, physical design, and networked communication.

## Our Mission

Sea Cadets Midwest STEM creates hands-on learning experiences that:

- **Inspire** the next generation of engineers and makers
- **Build** practical skills in firmware, electronics, and IoT
- **Develop** teamwork, problem-solving, and communication through project-based learning
- **Connect** students to career pathways in technology and defense

## Team-Based Learning

The program runs with **students organized into teams of 3**. Each team is assigned to a dedicated **Creation Station** - a physical workstation built around an Arduino Uno Q board with breadboard, sensors, components, and power supply. Equipment remains at the station throughout the program; teams build on their work day over day.

### Typical Team Roles During Labs

- **Student A:** Code author (typing in Arduino App Lab)
- **Student B:** Hardware technician (wiring, sensor placement, circuit building)
- **Student C:** Observer/documenter (reading instructions, verifying results, recording observations)

Roles rotate throughout each lab so all students get experience in every area.

## No Online Accounts Policy

**This program does NOT require students to create, register for, or log into any online accounts.**

This is a hard rule with no exceptions. The program serves minors in a residential setting, and requiring online accounts introduces unacceptable risks including privacy exposure, account security concerns, and terms of service compliance issues.

### What This Means

- All software tools are **pre-installed** on student Arduino Uno Q devices (no browser-based tools)
- All services (MQTT broker, dashboards) run on **local, isolated equipment**
- 3D modeling uses **FreeCAD**, which runs offline with no accounts needed
- Version control is **local-only** - no GitHub, GitLab, or cloud services
- Student boards connect to a **dedicated isolated Wi-Fi** for IoT traffic only
- No student workstation PCs are used - all development happens directly on the Arduino Uno Q

## Hardware Used

- **Arduino Uno Q** - Creation Station boards featuring a dual-brain architecture (Qualcomm Dragonwing + STM32 microcontroller) running Debian Linux with built-in BLE and an 8&times;13 RGB LED matrix
- **2004 I2C LCD displays** - Mounted at each station for real-time sensor feedback
- **Sensor kits** - LDRs, thermistors, potentiometers, buttons, LEDs, servos, ultrasonic sensors, PIR sensors, and breadboards
- **3D printers** - SOVOL SV06 ACE FDM printers for prototyping custom enclosures and parts
- **Local server VM** - Runs Mosquitto MQTT broker and Node-RED dashboard (instructor-managed)
